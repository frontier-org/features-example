üìã Getting Started

Frontier is a desktop application framework that allows you to build native windows applications using HTML, CSS, and JavaScript, with backend support for C (GCC), Python, and Batch scripts.

Quick Overview

  • Native Execution: Run C programs compiled with GCC for high-performance computing
  • Interpreted Scripts: Execute Python scripts for AI and data analysis
  • System Integration: Run Windows batch scripts for system operations
  • Cross-Platform UI: Build interfaces with HTML/CSS and JavaScript

üéØ Core Features

Three Execution Modes

NATIVE (GCC) ‚Üí High-performance C/C++ programs
INTERPRETED ‚Üí Python scripts and data analysis
SYSTEM ‚Üí Windows batch and shell operations

Window Management

Control window properties via meta tags in the HTML head:

<meta name="frontier-width" content="1200">
<meta name="frontier-height" content="800">
<meta name="frontier-resizable" content="true">

IPC Communication

Communicate with the backend using the Frontier API:

window.Frontier.dispatch(type, message)
window.Frontier.spawn(url, config)

‚öôÔ∏è Meta Tags Reference

Window Configuration

  • frontier-width - Set window width in pixels
  • frontier-height - Set window height in pixels
  • frontier-x - Horizontal position (use expressions)
  • frontier-y - Vertical position (use expressions)
  • frontier-min-width - Minimum window width
  • frontier-max-width - Maximum window width
  • frontier-resizable - Allow window resizing
  • frontier-maximizable - Show maximize button
  • frontier-persistent - Save/restore window state

Security & Navigation

  • frontier-allowed-internal - Whitelist for internal navigation
  • frontier-ignore-global-security - Bypass security whitelist
  • frontier-allowed-browser - Whitelist for browser navigation

üí° Tips & Best Practices

  • Use frontier-persistent="true" to save window state between sessions
  • Use expressions like (screen_w - win_w) / 2 for centering
  • Separate security: ignore_global_security for specific windows only
  • Test IPC communication: Check for window.ipc availability
  • Use meta tags for configuration instead of hardcoding values
  • Always provide fallback UI for standalone/web environments
